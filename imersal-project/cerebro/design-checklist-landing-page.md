# Checklist de Design — Landing Page

## Antes de Publicar

Marcar cada item antes de ir ao ar. Uma LP com design fraco mata conversão mesmo com tráfego caro.

---

## Above the Fold (Primeiro Scroll)

- [ ] Hero com headline clara e visível sem scroll
- [ ] CTA (botão) visível acima do fold em mobile E desktop
- [ ] Imagem ou vídeo que suporta a mensagem (não decorativa)
- [ ] Contraste de texto adequado (mínimo 4.5:1 WCAG AA)
- [ ] Sub-headline explica o produto/serviço em 1 frase
- [ ] Nenhum menu de navegação que desvie o usuário

## Mobile First

- [ ] Testado em iPhone SE (tela menor, 375px)
- [ ] Testado em iPhone 14 Pro (375px @3x)
- [ ] Testado em Android (360px)
- [ ] Botão CTA tem pelo menos 44px de altura (área de toque)
- [ ] Textos não menores que 16px (evita zoom automático no iOS)
- [ ] Imagens não cortadas no mobile
- [ ] Formulários com tipo de input correto (tel, email, number)

## Performance

- [ ] Loading < 3 segundos em 3G (testar no Chrome DevTools)
- [ ] Imagens comprimidas (WebP, máx 200kb por imagem)
- [ ] Sem vídeo autoplay pesado no hero (usar thumbnail + play)
- [ ] Google PageSpeed > 80 no mobile
- [ ] Pixel Meta instalado e disparando
- [ ] Google Analytics / GA4 configurado

## Conteúdo e Hierarquia

- [ ] Headline > Sub > Body (tamanhos decrescentes, hierarquia clara)
- [ ] Espaçamento generoso entre seções (não parece jornal)
- [ ] Prova social (depoimento, reviews, logos) acima do segundo fold
- [ ] FAQ com as objeções principais
- [ ] CTA repetido no final da página

## Formulário / CTA

- [ ] Formulário pede apenas o essencial (nome + telefone, no máximo)
- [ ] Botão tem cor contrastante com fundo (não cinza)
- [ ] Texto do botão é ação ("Quero meu PocketSafe", não "Enviar")
- [ ] Confirmação após submit (não fica em branco)
- [ ] Redirect para página de obrigado com próximos passos

## Depois do Ar

- [ ] Heatmap instalado (Hotjar ou Microsoft Clarity — gratuito)
- [ ] Meta Event Manager mostrando PageView e Lead
- [ ] Testar fluxo completo em mobile como usuário real
