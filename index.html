Bloco 01 — Fundamentos
Engenharia de prompt morreu. Contexto é o novo prompt.
O que é IA, o que é o Claude, e por que prompt mágico não existe mais

O que é LLM?

LLM = Large Language Model (Modelo Grande de Linguagem). É o "motor" por trás de tudo que vocês chamam de "inteligência artificial".

Assim como um motor de carro pode estar num Civic ou numa Ferrari — o mesmo tipo de tecnologia está em produtos diferentes:

  • ChatGPT é o produto da OpenAI. É o mais famoso.
  • Claude é o produto da Anthropic. É o que eu uso.
  • Gemini é o produto do Google.
  • Copilot é o produto da Microsoft.

E o Gamma? Canva AI? Notion AI? Também usam LLMs por baixo, mas empacotados pra uma função específica. O Gamma faz apresentação, o Canva faz design, o Notion AI faz texto dentro do Notion.

Quando você usa o Gamma pra fazer um slide, ele tá usando uma LLM por trás. A diferença é que no Claude ou ChatGPT você conversa direto com o motor — sem limite de pra que usar.

Mesmo preço: Pro de ambos é ~$20/mês (~R$115). Gratuito de ambos é gratuito.

O que o ChatGPT tem — e onde ele para

Antes de falar do Claude, é justo reconhecer o que o ChatGPT faz bem:

  • GPTs — são os "agentes" do ChatGPT. Você cria um chatbot especializado (nutricionista, copywriter, analista) e ele fica disponível pra usar. Legal, mas é uma conversa isolada — não conecta em nada, não executa nada fora do chat.
  • Canvas — uma tela de edição lado a lado com o chat. Útil, mas limitado a texto e código.
  • Projetos — organiza conversas por tema. Mas aqui tá o ponto: um chat não sabe o que você disse no outro. Os projetos do ChatGPT são pastas visuais, não inteligência compartilhada.
  • DALL-E / Sora — gera imagem e vídeo. O ChatGPT é superior aqui. Claude não gera imagem.

O grande problema: o ChatGPT é pergunta e resposta. Você manda mensagem, ele responde. Manda outra, responde de novo. Ele não executa coisas no mundo real — não cria arquivos no seu computador, não acessa suas ferramentas, não trabalha sozinho enquanto você faz outra coisa. É um chat inteligente. Só um chat.

A diferença fundamental No ChatGPT, você conversa COM a IA. No Claude (especialmente com Cowork e Claude Code), a IA trabalha PRA você. Essa é a mudança de paradigma.

Engenharia de prompt morreu

Antes: galera decorava fórmulas ("aja como um especialista em X com 20 anos de experiência..."). Tinha até "engenheiro de prompt" como profissão.

Hoje: o modelo é inteligente o suficiente — o que importa é CONTEXTO, não fórmula.

O estudo de Stanford que provou isso

Pesquisadores da Universidade de Stanford fizeram um experimento: colocaram engenheiros de prompt profissionais e pessoas leigas (sem experiência nenhuma com IA) para resolver as mesmas tarefas usando LLMs.

Resultado: os leigos tiveram resultados iguais ou melhores. Por quê? Porque falavam naturalmente — davam contexto real, explicavam o problema de verdade. Os engenheiros ficavam tentando "hackear" o modelo com templates e fórmulas.

A conclusão: não existe prompt mágico. Existe contexto bom.

Contexto = quem você é, o que quer, por que, pra quem, com que restrições.

Regra de ouro Fala com a IA como falaria com um estagiário muito inteligente que não te conhece. Quanto mais contexto, melhor o resultado.

Os 3 modos do Claude

O Claude tem 3 modos de trabalho. Cada um faz coisas diferentes:

Chat — pra entender

Conversas rápidas, brainstorming, perguntas. Como falar com um colega. Inclui modo voz e captura de tela.

Cowork — pra conectar

Tarefas complexas e autônomas. O Claude trabalha sozinho — pesquisa, analisa documentos, gera relatórios, cria arquivos. Você delega e ele entrega. (Vamos explicar em detalhe no Bloco 03.)

Code — pra construir

O próximo nível. Skills customizadas, agentes, times de I.A., apps, dashboards. Tudo pelo terminal. (Bloco 06.)

O vocabulário que você vai ouvir daqui pra frente

Ao longo dessa aula vão aparecer 4 termos o tempo todo. Antes de continuar, precisa ficar claro o que cada um é — senão vira sopa de sigla.

Skill — o manual de instruções

Imagine que você contratou alguém novo. Antes de ele começar, você explica: o que é o trabalho, como fazer, o tom, o que nunca fazer, como é um bom resultado. Isso — essa explicação, por escrito — é uma skill.

É um arquivo de texto que ensina o Claude a executar uma tarefa específica do mesmo jeito toda vez. Não é código. É texto puro. Qualquer pessoa sabe criar.

Exemplo: uma skill de "post pro Instagram" diz: público é tal, tom é direto, formato é carrossel de 8 slides, slide 1 sempre com gancho forte, CTA no final. Com essa skill instalada, toda vez que você pede um post, sai no padrão — sem precisar explicar de novo.

Plugin — o pacote de ferramentas pronto

Plugin é como instalar um app no celular. Antes de instalar, o celular não faz aquilo. Depois, faz.

Um plugin é um pacote de várias skills e ferramentas já montado por alguém, pronto pra usar. Você instala e o Claude ganha uma capacidade nova inteira.

Exemplo: o plugin de Marketing já vem com pesquisa de mercado, análise de concorrente, definição de posicionamento e criação de copy. Tudo junto, pronto. Você não precisa criar cada skill na mão.

Conector (MCP) — o crachá de acesso

Seu funcionário novo é inteligente, mas no primeiro dia não tem acesso a nada. Não entra no sistema, não vê a planilha, não abre o Drive.

O conector é o crachá. Conecta o Claude em uma ferramenta que você já usa — Google Drive, Notion, Meta Ads, Slack — e dá acesso pra ele trabalhar lá dentro.

Exemplo: com conector do Google Drive, o Claude entra no seu Drive, lê seus documentos, edita suas planilhas. Sem conector, você teria que copiar e colar tudo na conversa.

Agente — o funcionário autônomo

Agente é um Claude que combina várias skills e trabalha sozinho do início ao fim. Você dá a missão e ele entrega o resultado pronto — sem ficar perguntando a cada passo.

Exemplo: um agente copywriter recebe o briefing, pesquisa concorrentes, escreve headline, body copy, CTAs e e-mail de follow-up. Entrega tudo formatado.

Coisas que o Claude gratuito faz (e que você provavelmente não sabe)

A maioria das pessoas acha que IA é só "fazer pergunta e receber resposta". O Claude gratuito vai muito além. Aqui vão coisas que você faz na mão hoje e que ele resolve em segundos:

Traduzir documento gigante com cláusulas

Tem um contrato em inglês com 40 páginas cheio de cláusula jurídica? Sobe o PDF e pede pra traduzir. Ele mantém a formatação, entende os termos técnicos e traduz tudo — não é Google Translate.

Prompt

Traduza este contrato para português brasileiro. Mantenha a estrutura de cláusulas, numeração e termos jurídicos adequados ao direito brasileiro.

Ler contrato e achar pontos críticos

Você recebeu um contrato com uma porrada de cláusula e não quer ler tudo? Sobe o arquivo e pergunta:

Prompt

Analise este contrato e me diga: 1. Quais são as 5 cláusulas mais importantes? 2. Tem alguma cláusula que me prejudica ou é abusiva? 3. Quais são os prazos e multas? Resumo executivo no final.

Fazer ata de reunião

Cola a transcrição de uma reunião e ele extrai: decisões tomadas, pendências, responsáveis e próximos passos.

Escrever e-mail formal difícil

Aquele e-mail que você passa 30 minutos pensando no tom? Dá o contexto e ele escreve em 10 segundos.

Montar apresentação

Descreve o tema e ele estrutura a apresentação inteira — slides, tópicos, narrativa. Pede como artefato e ele gera visual na conversa.

Comparar propostas de fornecedores

Sobe 3 PDFs de propostas, ele monta tabela comparativa com preço, prazo, escopo e recomendação.

Organizar dados bagunçados

Cola uma lista bagunçada de qualquer coisa e ele estrutura em tabela limpa.

Tudo isso é no gratuito Você não precisa pagar pra começar a resolver problemas reais. O limite é a quantidade de mensagens por dia, não a qualidade.

Projetos — a feature mais subestimada

No Claude, você cria projetos. Cada projeto tem instruções fixas (quem você é, como quer as respostas) e pode ter arquivos anexados.

A diferença real pro ChatGPT

O ChatGPT também tem "projetos". Mas lá, uma conversa não sabe o que você disse na outra dentro do mesmo projeto. É só uma pasta visual.

No Claude, funciona de verdade. As instruções do projeto estão sempre ativas em qualquer conversa dentro dele. O contexto persiste. Cria projeto pra tudo que você faz de forma sistemática.

O que é uma ação?

Toda vez que o Claude faz algo — qualquer coisa — ele está executando uma ação. Ação é o átomo, a unidade mínima de trabalho.

Quando você pede "resume esse texto" — é uma ação. "Pesquisa na web" — ação. "Lê esse PDF" — ação. "Manda um e-mail" — ação.

Sozinha, uma ação é genérica. O Claude faz, mas sem contexto do seu negócio, sem seu tom, sem suas regras. Funciona, mas é raso.

A progressão: de ação genérica a máquina

Aqui é onde tudo começa a fazer sentido. Cada conceito que você vai aprender nos próximos blocos adiciona uma camada em cima da ação:

Ação — o Claude faz algo (genérico)
↓
Ação + Treinamento = Skill — faz do jeito certo, toda vez
↓
Skill + Conector = Capaz — faz do jeito certo, com acesso às suas ferramentas
↓
Plugin = Pacote completo — skills + conectores + ações, tudo empacotado

Exemplo concreto: gerando leads

Vamos ver como a mesma tarefa fica em cada nível:

Sozinha, cada peça é útil. Juntas, viram uma máquina.

Bloco 09 — Cowork & Claude Code
O que é Cowork: o Claude vira funcionário.
Ele trabalha sozinho enquanto você faz outra coisa

O problema do chat (gratuito ou pago)

No modo Chat, o Claude conversa com você. Você manda mensagem, ele responde, você manda outra. É ping-pong. Pra perguntas rápidas, ótimo. Mas pra tarefas grandes, você fica preso respondendo a cada passo.

O que muda no Cowork

No Cowork, você dá uma tarefa inteira e o Claude sai trabalhando sozinho. Ele pesquisa na internet, lê seus documentos, cria arquivos, monta relatórios — sem precisar da sua intervenção a cada passo. Você delega e vai fazer outra coisa.

O que o Cowork faz que o ChatGPT não faz

  • Cria arquivos reais no seu computador — Excel com fórmulas, Word formatado, PowerPoint com slides, PDFs prontos. Não é texto na conversa — é arquivo salvo na sua pasta.
  • Pesquisa na web em tempo real — com fontes. Não é conhecimento antigo.
  • Acessa ferramentas externas — Google Drive, Meta Ads, Notion, Slack — via plugins e conectores (vamos explicar nos próximos blocos).
  • Trabalha de forma autônoma — você delega tarefa complexa e ele executa sozinho, passo a passo.

No ChatGPT, mesmo no pago, você fica preso no ping-pong. Não cria arquivos, não conecta em ferramentas, não tem autonomia.

Por que criar sua própria skill?

As skills prontas (find-skills, copywriting, etc.) são ótimas pra começar. Mas o verdadeiro poder é quando você cria a sua — com seu processo, seu tom, suas regras, seu formato de entrega.

Ninguém conhece seu negócio melhor que você. Uma skill personalizada transforma o Claude num funcionário que trabalha exatamente do jeito que você precisa.

Criando uma skill do zero

Uma skill é um arquivo de texto (.md) com instruções. Não é código. Se você sabe escrever um manual pra um estagiário, sabe criar uma skill.

Estrutura básica

Exemplo — skill-proposta-comercial.md--- name: Proposta Comercial description: Gera propostas comerciais profissionais a partir de briefing do cliente. --- # Skill: Proposta Comercial ## Objetivo Criar propostas comerciais formatadas e profissionais a partir das informações do cliente. ## Processo 1. Pergunte: nome do cliente, empresa, problema principal, orçamento estimado. 2. Pesquise o segmento do cliente pra contextualizar. 3. Estruture a proposta com: capa, diagnóstico, solução, escopo, investimento, prazo, próximos passos. 4. Tom: profissional, direto, sem jargão desnecessário. 5. Se não tem, pergunte. - Sempre inclua seção de "Por que nós" com diferenciais. - Investimento sempre com 3 opções (básico, recomendado, premium). - Prazo realista — não prometa menos de 2 semanas pra nada. ## Formato de entrega Arquivo HTML com design profissional, pronto pra imprimir ou enviar como PDF.

O que é um agente?

Um agente é um Claude treinado e autônomo. Você dá uma missão, ele executa do início ao fim, sozinho. Não é ping-pong — é delegação.

Skill vs Agente

Skill: você + Claude juntos. Ele segue o manual, mas você supervisiona e aprova cada passo. É colaboração.

Agente: ele trabalha sozinho. Recebe a missão e entrega o resultado pronto. É delegação.

Por que um agente e não só uma skill?

A pergunta mais importante: "por que não resolvo isso só com uma skill?"

E por que não só um conector?

O conector dá acesso, mas não sabe o que fazer com ele. É como dar a chave do carro pra alguém que não sabe dirigir.

  • Só conector: o Claude acessa seu Google Drive. E aí? Ele não sabe o que procurar, como analisar, em que formato entregar.
  • Conector + Skill: acessa o Drive E sabe analisar contratos. Melhor.
  • Conector + Skill + Agente: acessa o Drive, analisa os contratos, compara com a versão anterior, gera relatório de diferenças, e te manda por e-mail. Isso é elite.

Exemplos reais de agentes

Agente de atendimento

Recebe pergunta do cliente, busca na base de conhecimento, responde no tom certo, escala pra humano se não souber.

Resultado: atendimento 24/7 sem contratar ninguém.

Por que não basta skill? Porque ele precisa decidir se sabe responder ou se escala. Precisa de autonomia.

Agente de pesquisa de mercado

Recebe "analisa o mercado de X", abre 15 sites, lê relatórios, cruza dados, identifica tendências, e entrega um documento de 10 páginas com gráficos.

Resultado: pesquisa que levaria dias, pronta em minutos.

Por que não basta skill? São dezenas de decisões no caminho — quais fontes usar, o que é relevante, como cruzar. Sem autonomia, você teria que aprovar cada passo.

Agente de conteúdo semanal

Todo domingo ele roda: lê seus rascunhos, pesquisa trending, escreve 5 roteiros de carrossel, 3 ideias de Reels, legenda e hashtags. Segunda de manhã você tem tudo pronto.

Resultado: conteúdo de uma semana em 30 minutos.

Por que não basta skill? Porque ele roda sozinho, sem você pedir. Ele toma as decisões de tema e formato baseado no que tá trending.

Às vezes você nem pede — o Claude Code identifica que a tarefa precisa de autonomia e cria um agente automaticamente. Exemplo real:

O que aconteceu: ele decidiu sozinho que precisava de um agente pra executar isso de forma eficiente. Você não configurou nada — ele orquestrou por conta própria.

O que é o CLAUDE.md?

É um arquivo de texto que funciona como manual de instruções pro Claude. Quando o Claude abre um projeto que tem um CLAUDE.md na raiz, ele lê automaticamente e segue as instruções.

Inteligência global = o Claude sempre sabe quem é você, como pensa, como quer ser atendido (via CLAUDE.md global).

Inteligência por projeto = quando trabalha no PocketSafe, sabe que é uma tag Bluetooth com logo de bulldog. Quando trabalha na New Bird, sabe que é uma agência de marketing. Cada projeto tem seu contexto.

Na prática Quando você abre o Claude Code numa pasta que tem CLAUDE.md, ele já chega sabendo tudo. É como um funcionário que leu o briefing antes de começar.

Meu sistema de cérebro externo

O problema: eu não parava de receber conteúdo. Todo dia era print no Instagram, link de YouTube, ideia no banho, trecho de podcast, post interessante. Tudo ia pra um buraco negro — salvava e nunca mais achava.

A solução: criei um recebedor universal de informação que processa e guarda no lugar certo automaticamente. Usando a API do Claude.

Como funciona

  1. Captura (celular ou computador)

    Vejo algo interessante? Mando pro cérebro. Print, link, texto, áudio — qualquer formato. Vai pra uma pasta de entrada (inbox) via iCloud.

  2. Roteamento automático

    Um script Python (router.py) monitora a inbox. Quando algo chega, ele lê o frontmatter do arquivo (tipo: insight/print/youtube/instagram/audio, projeto: new-bird/pocket-safe/etc) e move pra pasta correta automaticamente.

  3. Processamento inteligente

    Pra links de YouTube e Instagram, o sistema extrai o conteúdo completo. Pra áudios, transcreve com Whisper. Pra prints, salva a imagem e cria a nota com contexto.

  4. Tudo organizado por setor

    Marketing vai pra conhecimento/marketing. Insight de produto vai pro projeto certo. Framework novo vai pra playbook. Sem eu fazer nada.

Bloco 19 — Ecossistema
Por que não pago nenhum SaaS de IA — e o que construí no lugar.
Manus, Lovable, Emergent, Cursor, Zapier — o que são, o que fazem, e por que eu não preciso

Toda semana aparece uma ferramenta nova prometendo "IA que faz tudo por você". O marketing é bom, o vídeo de demo impressiona, e dá vontade de assinar. Mas quando você entende o que essas ferramentas realmente fazem por baixo dos panos, percebe que construiu coisa mais poderosa no terminal.

Vou explicar cada uma — o que faz, como faz, e o que eu criei que substitui.

Manus — $20/mês

O que é

Um agente autônomo que navega a web, executa código e entrega "deliverables" — PDFs, planilhas, sites simples. Você pede "pesquisa os 10 melhores CRMs do Brasil e me entrega uma planilha comparativa" e ele faz sozinho: abre o navegador, pesquisa, compara, monta o arquivo.

Mito: Manus não é da Meta. É de uma empresa chinesa (Monica AI). Não tem acesso especial a APIs do Instagram ou Facebook. Quando você vir esse tipo de claim, desconfie.

O que ele faz por baixo

Manus é basicamente um navegador controlado por IA + gerador de arquivos. Ele abre sites, lê conteúdo, processa, e exporta. É isso. Não tem memória entre sessões, não conhece seu projeto, não se conecta nas suas ferramentas. Começa do zero toda vez.

O que eu construí no lugar

Isso é o Cowork — o modo do Claude que trabalha sozinho em segundo plano. Ele faz exatamente isso: pesquisa, processa, entrega. Mas com uma diferença brutal: ele conhece meu projeto, minhas preferências, meu histórico. Não começa do zero.

E quando preciso de mais poder, vou pro Claude Code no terminal. Aí não é só navegação — é acesso real ao sistema. O Claude lê meus arquivos, edita código, commita no GitHub, deploya no Vercel. O Manus não chega perto disso.

Pra pesquisa e scraping, tenho Firecrawl — que não só navega como baixa código fonte inteiro de sites, extrai design systems, preenche formulários. E o Rube conecta em 1000+ apps (Slack, Gmail, Notion, Jira, Google Sheets) direto no Claude — sem precisar de um "agente" separado navegando pela web.

Lovable e Emergent — $20/mês cada

O que são

Geradores de app por prompt. Você descreve "quero um dashboard de vendas com gráficos e login" e eles geram o app inteiro — frontend, backend, banco de dados. Visual bonito, deploy rápido. O Lovable usa React + shadcn + Supabase. O Emergent (YC S24, 3M+ usuários) faz web e mobile.

O que fazem por baixo

Pegam seu prompt, geram código com IA, renderizam num preview visual, e hospedam na plataforma deles. Funciona bem pra protótipo. Mas tem dois problemas grandes:

  • O código fica preso. Exportar até dá, mas não é seamless. Você depende da plataforma.
  • Sem contexto. Cada geração começa do zero. Não sabe nada sobre sua marca, seus componentes, seus padrões.

O que eu construí no lugar

Claude Code + shadcn + 21st.dev + Vercel. Eu peço "cria uma landing page pro PocketSafe com hero, features e checkout" e ele gera — mas no meu repositório, com meus componentes, respeitando minha brand guideline, e publica no meu domínio com git push.

A diferença é que o Claude já conhece meu projeto. Ele sabe que o PocketSafe usa um bulldog como logo, que a paleta é X, que os componentes seguem Y padrão. Não preciso explicar nada — é contexto acumulado. Por isso a evolução é tão cavalar: cada coisa que eu construo torna a próxima mais rápida.

No Lovable, eu teria que re-explicar tudo isso toda vez. 100 créditos por mês no Emergent? Eu gero sem limite.

Cursor e Windsurf — $20/mês

O que são

IDEs (editores de código) turbinadas com IA. O Cursor é um VS Code modificado que entende seu código e sugere edições. O Windsurf é parecido. Você seleciona um trecho, pede pra refatorar ou adicionar feature, e ele edita inline.

O que fazem por baixo

Leem seu código, mandam pro modelo de IA junto com contexto do arquivo, e aplicam as sugestões de volta. Autocomplete sofisticado + chat que entende seu projeto.

O que eu construí no lugar

Claude Code no terminal faz tudo isso — e mais. Ele não só edita código: ele roda testes, commita, deploya, acessa banco de dados, gera imagens, pesquisa na web, cria apresentações. É a diferença entre ter um editor inteligente e ter um funcionário full-stack morando no seu terminal.

E o mais importante: o Claude Code tem acesso a todo o ecossistema. Skills de copywriting, conectores com Figma, geração de imagem com Nano Banana, acesso ao Supabase — tudo no mesmo lugar. O Cursor é só código. O Claude Code é código + tudo mais.

Jasper e Copy.ai — $39-49/mês

O que são

Ferramentas de copywriting com IA. Você escolhe um template (e-mail de vendas, post de Instagram, headline de landing page), preenche campos, e elas geram o texto.

O que fazem por baixo

Templates pré-definidos + LLM. É um ChatGPT com prompts prontos e uma interface bonita em volta. O "segredo" deles é a curadoria de prompts — que é exatamente o que uma skill faz.

O que eu construí no lugar

Skill de copywriting + cérebro no Obsidian. Minha skill de copy não é um template genérico — ela tem os frameworks que eu estudei (AIDA, PAS, Before-After-Bridge), os exemplos que eu curei, o tom de voz da minha marca. E o cérebro tem arquivos inteiros sobre hooks, estrutura de landing page, sequência de e-mail.

Quando peço "escreve o e-mail de lançamento do PocketSafe", o Claude puxa tudo isso automaticamente. O Jasper me pediria pra preencher 6 campos de um template genérico.

O que é

O que faz por baixo

Webhooks + APIs. Quando um trigger dispara, o Zapier chama a API do próximo app na cadeia. Funciona, mas é limitado: fluxos lineares, lógica simples, e você paga por "zap" (execução).

O que eu construí no lugar

Eu não monto fluxos visuais — eu falo "manda pro João no Slack o resumo da reunião e atualiza a planilha de leads" e ele faz. Linguagem natural, sem arrastar bloquinhos.

Por que a evolução é tão rápida

Quando você usa Lovable, Emergent, ou Manus, cada projeto começa do zero. Você descreve tudo de novo, configura tudo de novo, e o resultado é genérico.

Quando você constrói o ecossistema no terminal, acontece algo diferente: cada projeto torna o próximo mais poderoso.

  1. Contexto acumula. O Claude aprende sua marca, seus padrões, suas preferências. O PocketSafe ensinou ele a respeitar brand guidelines. O Leeven ensinou a construir sistemas complexos. Cada projeto deixa rastro.
  2. Skills evoluem. A skill de copywriting que criei há 3 meses já foi refinada 20 vezes. É um ativo que melhora com uso — não um template estático.

Preciso pagar pra começar?

Não. O gratuito já resolve muita coisa — tradução, análise, brainstorming, conteúdo. O Pro (R$115/mês) libera Cowork e Claude Code.

Preciso saber programar?

Não. Skills são texto puro. Você conversa em português. Se sabe escrever um manual, sabe criar uma skill.

Funciona no Windows?

Sim. Desktop App no Windows 10+. Claude Code em Mac, Windows e Linux.

Qual a diferença entre Plugin e Skill?

Plugin = pacote de ferramentas pronto (como um app). Skill = manual de instruções pra uma tarefa (como uma receita).

Posso usar ChatGPT em vez do Claude?

Skills funcionam em várias ferramentas. Mas agentes, squads, MCPs e subagentes são exclusivos do Claude Code.

O Claude usa meus dados pra treinar?

Sim, por padrão. Desative: claude.ai → Settings → Privacy → desmarque "Improve Claude for everyone".

Meu PC é lento — vai travar?

O Claude roda nos servidores da Anthropic, não no seu PC. Só precisa de navegador ou terminal funcional.
